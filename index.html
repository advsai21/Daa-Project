01 — About

Why Graph
Colouring?

Traditional manual seating is error-prone and time-consuming for large cohorts. By treating the exam hall as a graph, we turn a logistics problem into a well-studied mathematical problem with provable correctness guarantees.

■ The Core Problem
Adjacent seats must be assigned to students from different branches, preventing possible malpractice. This is precisely the Graph Colouring constraint: no two connected nodes may share the same colour (branch).
■ The Graph Model
Each seat is a vertex. Seats sharing a physical boundary (8-directional) are connected by edges. We then colour this graph with exactly 7 colours, one per branch, satisfying the adjacency constraint globally.
■ The Guarantee
Post-assignment, the system runs a validation pass over every edge. If any adjacent pair shares a branch, a conflict is raised. For standard hall grids, zero conflicts are guaranteed by the algorithm.

03 — Algorithm

Welsh-Powell Step by Step

A greedy graph colouring heuristic that runs in O(V+E) time — efficient enough to colour 500+ seats in under 2 seconds.

01
Build the Adjacency Graph
Each seat (i,j) becomes a vertex. Add edges for all 8-directional neighbours within hall bounds.
02
Sort by Degree (Descending)
Order all vertices by the number of adjacent seats. Interior seats (degree 8) are coloured first, reducing backtracking.
03
Greedy Colour Assignment
Assign each vertex the smallest colour (branch ID) not already used by any of its coloured neighbours.
04
Map Colours → Students
Collect seats per colour. Shuffle the matching branch's student list. Assign one-to-one. Every student gets exactly one seat.
05
Validate & Export
Walk every edge and assert no same-branch pair. Generate printable PDF seating chart and individual seat slips.
# Welsh-Powell Graph Colouring # O(V + E) — handles 500 seats < 2s def welsh_powell(graph): # Step 1: Sort by degree descending vertices = sorted( graph.nodes(), key=lambda v: graph.degree(v), reverse=True ) colour_map = {} # vertex → branch_id BRANCHES = ['CSE','ECE','EEE','MECH', 'CIVIL','CHEM','BIOTECH'] for v in vertices: # Colours used by neighbours used = { colour_map[n] for n in graph.neighbors(v) if n in colour_map } # Pick lowest available branch for branch in BRANCHES: if branch not in used: colour_map[v] = branch break return colour_map # seat → branch def validate(graph, colour_map): conflicts = [] for u, v in graph.edges(): if colour_map[u] == colour_map[v]: conflicts.append((u, v)) return conflicts # [] = success
